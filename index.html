Introduction

Introduction: About

  • This is a web application that leverages GenAI for summarization.
  • The application allows to summarize, share and search articles.

Introduction: Use Cases

  • Facilitating the internal sharing of GenAI documents within EPAM.
  • Assisting in the storage and management of documents from journal clubs.
  • Supporting universities in the effective managing of their academic repositories.
  • Assisting retail industries in managing their user manual repositories.

Architecture

Architecture

  • Python as main programming language.
  • Langchain as the main framework.
  • FastAPI and streamlit for the web app.
  • MongoDB and ChromaDB as databases.
  • SSO with azure.

Next Steps

Next Steps: Unstable Features

  • Clustering-based RAG summarization.
  • Instead of searching the most related chunks to the title, this technique would allow to have diverse clusters that provide a better description of a document.

Next Steps: Unstable Features

  • RAG-based summary edition with chat assistant.
  • Manage chat memory for editions.
  • The chat assistant would retrieve relevant chunks and user's mesasges.

Next Steps: Future Work

  • Web scraping with SSO authentication.
  • Chat assistant for a better interaction with newsfeed articles.
  • Query document's metadata with natural language queries.
  • A general GenAI web API as a stand-alone product.
  • Handle PII data.

Conclusions

Conclusions

  • RAG provides a fast and low-cost solution for abstractive summarization.
  • Vector databases are effective in the implementation of semantic search engines.
